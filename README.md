# VLM Game Agent

一个基于大视觉语言模型（VLM）的 GUI 自动化 Agent 系统，能够让 AI 接管指定窗口（通常是游戏窗口）并通过视觉理解自主完成用户设定的目标。

---

## 核心能力

- **窗口捕获**：持续截取指定窗口的画面，作为 VLM 的实时视觉输入。
- **GUI 操作 Agent**：将 VLM 的输出解析为具体的鼠标、键盘操作，实现对窗口的自动化控制。
- **目标驱动**：用户设定一个高层目标，Agent 会自主规划、执行、观察反馈，直到目标达成为止。
- **人机协作**：执行过程中，用户可随时介入并给出指令，Agent 会结合当前状态调整行为。
- **技能系统（Skills）**：支持 Agent 在任务过程中总结经验并自动生成可复用的 Skill；用户也可手动编写或安装 Skill，以扩展 Agent 的能力边界。
- **思考-执行同步（Pause & Resume）**：在 VLM 推理前暂停游戏画面，待 AI 思考并输出操作指令后再恢复游戏并执行，避免实时游戏中因推理延迟导致决策落后。

---

## 系统架构

### 1. 视觉输入层（Vision Input）
- 选定目标窗口（支持任意应用程序/游戏窗口）。
- 持续、高效地获取窗口截图，控制帧率以平衡实时性与 Token 消耗。
- 可选：对截图进行预处理（缩放、区域裁剪、画质压缩）。

### 2. Agent 决策层（Agent Core）
- **提示词工程（Prompt Engineering）**
  - 系统提示词：定义 Agent 的身份、能力边界、输出格式。
  - 任务提示词：将用户目标转化为可执行的步骤规划。
  - 操作提示词：将 VLM 的推理结果映射为标准化的 GUI 动作（点击、拖拽、按键等）。

- **记忆系统（Memory）**
  - **文件化存储**：指定一个记忆文件夹，所有记忆文件以 Markdown 格式存放，Agent 启动时自动加载。
  - **默认文件**：
    - `memories.md` —— 存放游戏机制、通用经验、失败教训等跨会话的长期知识。
    - `coordinates.md` —— 存放与坐标相关的关键位置信息（如按钮坐标、地图标记等）。
  - **用户可扩展**：用户可在记忆文件夹中新增任意 `.md` 文件，并在其中编写引导性内容，Agent 会将这些文件内容持续保持在上下文中，辅助决策与回忆。
  - **上下文保持**：所有记忆文件的内容经过 Token 预算后，作为系统上下文的一部分常驻在每次 VLM 请求中。

- **上下文管理 / 压缩（Context Compression）**
  - **纯文本上下文**：历史视觉截图不直接进入大模型上下文，仅保留当前帧图像；历史记录以文本形式维护（如操作日志、状态摘要、关键事件描述）。
  - **阈值触发**：设定 Token 数量或上下文长度阈值，当累计文本超过阈值时触发压缩。
  - **可配置压缩模型**：摘要压缩任务可由独立配置的轻量模型（或本地模型）承担，无需占用主 VLM 的调用额度与上下文窗口。

- **技能系统（Skills）**
  - **Skill 定义**：一段可复用的结构化知识，包含触发条件、执行流程、预期结果与异常处理（如“自动寻路”、“连招释放”、“背包整理”）。
  - **自动发现**：Agent 在多次完成相似任务后，自动提取共性步骤并生成新的 Skill，纳入自身工具箱。
  - **手动安装**：用户或开发者通过文件/接口手动导入 Skill，实现快速定制与社区共享。
  - **动态调用**：Agent 根据当前场景与目标，自动判断应该调用哪些 Skill，或将多个 Skill 组合为更复杂的策略。

### 3. 时停控制层（Pause Controller）
- **支持快捷键暂停的游戏（软暂停）**：用户需在配置中预先指定该游戏的暂停快捷键（如空格、Esc）；截图后 Agent 立即发送该快捷键冻结画面，为 VLM 争取思考时间，待操作指令生成后先执行动作，再发送恢复快捷键。
- **不支持快捷键暂停的游戏（硬暂停）**：
  - **失焦暂停**：将焦点切出游戏窗口，利用多数游戏在失焦时自动暂停的机制；执行前再切回焦点。
  - **硬暂停（进程级）**：若游戏内无暂停快捷键，则通过挂起游戏进程（如 Windows `NtSuspendProcess`）强制冻结游戏状态，待 AI 输出操作后再恢复进程，确保画面绝对静止。
- 该层确保“冻结 → 思考 → 执行 → 恢复”的原子性，防止 AI 在思考期间游戏状态发生漂移。

### 4. 动作执行层（Action Execution）
- 接收 Agent 输出的结构化动作指令（如 `click(x, y)`、`press('space')`）。
- 调用系统 API 或自动化框架（如 `pyautogui`、`pydirectinput`）精确执行。
- 执行后自动截图，形成“操作 → 观察”的闭环反馈。

### 5. 远程监看层（WebUI）
- 提供轻量级 Web 界面，支持远程访问。
- 实时显示当前窗口画面、Agent 的思考过程、历史操作流水。
- 支持用户通过 WebUI 下发即时指令或修正 Agent 行为。

## 工作流程

1. **初始化**：用户选择目标窗口，Agent 建立连接并开始持续截图。
2. **目标设定**：用户输入高层目标（例如：“击败第一个 BOSS”、“收集 10 个材料”）。
3. **循环执行**：
   - 截图 → 触发时停控制层暂停游戏；
   - 当前帧图像 + 文本历史送入 VLM；
   - VLM 结合记忆与历史进行推理，输出下一步操作；
   - 时停控制层恢复游戏；
   - 动作执行层执行操作并等待画面变化；
   - 检查目标是否达成，未达成则继续循环。
4. **人工介入**：用户在任意时刻通过 WebUI 暂停 Agent 或发送指导消息，Agent 会将人工输入纳入后续推理。
5. **任务结束**：目标达成后，Agent 停止操作并生成任务总结，同时将新习得的知识写回记忆文件夹中的对应 Markdown 文件。

## 技术栈

| 模块 | 推荐方案 | 理由 |
|------|---------|------|
| 窗口捕获 | `mss` + `pygetwindow` | `mss` 跨平台且性能极高（纯内存截图）；`pygetwindow` 获取窗口句柄与坐标。 |
| GUI 自动化 | `pydirectinput` + `pyautogui` | `pydirectinput` 绕过 Windows 输入保护，适合操控游戏；`pyautogui` 作为通用 fallback。 |
| VLM 调用 | `openai` / `anthropic` SDK + `httpx` | 官方 SDK 稳定；`httpx` 作为异步 HTTP client，方便后续接入本地 vLLM / Ollama。 |
| 图像预处理 | `Pillow` | 足够完成缩放、裁剪、编码压缩；需要模板匹配时再引入 `opencv-python`。 |
| 时停控制（软暂停） | `pynput` | 发送全局快捷键稳定，不受游戏窗口焦点限制。 |
| 时停控制（硬暂停） | `pywin32` + `ctypes` | 调用 `NtSuspendProcess` 挂起/恢复目标进程。 |
| 配置管理 | `pydantic-settings` | 类型安全、支持 `.env` 与环境变量覆盖，适合管理 API Key、暂停键、进程名等。 |
| 异步调度 | `asyncio` + `httpx` | Agent 主循环（截图 → 暂停 → HTTP 请求 → 执行）全链路异步，避免阻塞。 |
| 日志 | `loguru` | 零配置、支持结构化日志与文件轮转，比标准库 `logging` 更省心。 |
| WebUI 后端 | `FastAPI` + `websockets` | FastAPI 原生支持 WebSocket，用于向前端实时推送画面与 Agent 思考流。 |
| WebUI 前端 | `Vue 3` (CDN) | 无需构建步骤，直接 HTML 引入，配合原生 WebSocket 实现实时监看。 |
