# VLM Game Agent

一个基于大视觉语言模型（VLM）的 GUI 自动化 Agent 系统，能够让 AI 接管指定窗口（通常是游戏窗口）并通过视觉理解自主完成用户设定的目标。

---

## 核心能力

- **窗口捕获**：持续截取指定窗口的画面，作为 VLM 的实时视觉输入。
- **GUI 操作 Agent**：将 VLM 的输出解析为具体的鼠标、键盘操作，实现对窗口的自动化控制。
- **目标驱动**：用户设定一个高层目标，Agent 会自主规划、执行、观察反馈，直到目标达成为止。
- **人机协作**：执行过程中，用户可随时介入并给出指令，Agent 会结合当前状态调整行为。
- **技能系统（Skills）**：支持 Agent 在任务过程中总结经验并自动生成可复用的 Skill；用户也可手动编写或安装 Skill，以扩展 Agent 的能力边界。

---

## 系统架构

### 1. 视觉输入层（Vision Input）
- 选定目标窗口（支持任意应用程序/游戏窗口）。
- 持续、高效地获取窗口截图，控制帧率以平衡实时性与 Token 消耗。
- 可选：对截图进行预处理（缩放、区域裁剪、画质压缩）。

### 2. Agent 决策层（Agent Core）
- **提示词工程（Prompt Engineering）**
  - 系统提示词：定义 Agent 的身份、能力边界、输出格式。
  - 任务提示词：将用户目标转化为可执行的步骤规划。
  - 操作提示词：将 VLM 的推理结果映射为标准化的 GUI 动作（点击、拖拽、按键等）。

- **记忆系统（Memory）**
  - **短期记忆**：维护当前任务的关键上下文（如当前所在菜单、血量、道具数量）。
  - **长期记忆**：跨会话保留游戏机制、成功经验、失败教训，加速后续任务。

- **上下文管理 / 压缩（Context Compression）**
  - 维护最近 N 帧的视觉与操作历史。
  - 当上下文接近模型上限时，对早期历史进行摘要化压缩，保留关键事件而丢弃冗余截图。

- **技能系统（Skills）**
  - **Skill 定义**：一段可复用的结构化知识，包含触发条件、执行流程、预期结果与异常处理（如“自动寻路”、“连招释放”、“背包整理”）。
  - **自动发现**：Agent 在多次完成相似任务后，自动提取共性步骤并生成新的 Skill，纳入自身工具箱。
  - **手动安装**：用户或开发者通过文件/接口手动导入 Skill，实现快速定制与社区共享。
  - **动态调用**：Agent 根据当前场景与目标，自动判断应该调用哪些 Skill，或将多个 Skill 组合为更复杂的策略。

### 3. 动作执行层（Action Execution）
- 接收 Agent 输出的结构化动作指令（如 `click(x, y)`、`press('space')`）。
- 调用系统 API 或自动化框架（如 `pyautogui`、`pydirectinput`）精确执行。
- 执行后自动截图，形成“操作 → 观察”的闭环反馈。

### 4. 远程监看层（WebUI）
- 提供轻量级 Web 界面，支持远程访问。
- 实时显示当前窗口画面、Agent 的思考过程、历史操作流水。
- 支持用户通过 WebUI 下发即时指令或修正 Agent 行为。

---

## 工作流程

1. **初始化**：用户选择目标窗口，Agent 建立连接并开始持续截图。
2. **目标设定**：用户输入高层目标（例如：“击败第一个 BOSS”、“收集 10 个材料”）。
3. **循环执行**：
   - 截图 → 送入 VLM；
   - VLM 结合记忆与历史进行推理，输出下一步操作；
   - 执行操作并等待画面变化；
   - 检查目标是否达成，未达成则继续循环。
4. **人工介入**：用户在任意时刻通过 WebUI 暂停 Agent 或发送指导消息，Agent 会将人工输入纳入后续推理。
5. **任务结束**：目标达成后，Agent 停止操作并生成任务总结，同时更新长期记忆。

## 技术栈（待定）

| 模块 | 候选方案 |
|------|---------|
| 窗口捕获 | `mss`, `PIL.ImageGrab`, `pygetwindow`, Win32 API |
